labCA

  • labCA
    • Common Arguments and Return Values — Used by several labCA functions
    • Error Handling — labCA's error reporting mechanism
    • lcaClear — Clear / release (disconnect) channels.
    • lcaDebugOff — Switch the ezca library's debugging facility off.
    • lcaDebugOn — Switch the ezca library's debugging facility on.
    • lcaDelay — Delay execution of scilab or matlab for the specified time to handle channel access activity (monitors).
    • lcaGet — Read a number of m PVs, which may be scalars or arrays of different dimensions.
    • lcaGetAlarmLimits — Retrieve the alarm limits associated with a number of PVs.
    • lcaGetControlLimits — Retrieve the control limits associated with a number of PVs.
    • lcaGetEnumStrings — Retrieve the symbolic values of all ENUM states of a number of PVs.
    • lcaGetGraphicLimits — Retrieve the graphic limits associated with a number of PVs.
    • lcaGetNelem — Retrieve the element count of a number of PVs.
    • lcaGetPrecision — Retrieve the precision associated with a number of PVs.
    • lcaGetRetryCount — Retrieve / set the ezca library retryCount parameter (consult the ezca documentation for more information).
    • lcaGetStatus — Retrieve the alarm severity and status of a number of PVs along with their timestamp.
    • lcaGetTimeout — Retrieve / set the ezca library timeout parameter (consult the ezca documentation for more information).
    • lcaGetUnits — Retrieve the engineering units of a number of PVs.
    • lcaGetWarnLimits — Retrieve the warning limits associated with a number of PVs.
    • lcaLastError — This routine is a simple extension to scilab's lasterror which only allows a single error to be reported.
    • lcaNewMonitorValue — Check if monitored PVs need to be read, i.
    • lcaNewMonitorWait — Similar to [1]lcaNewMonitorValue but instead of returning the status of monitored PVs this routine blocks until all PVs have fresh data available (e.
    • lcaPut — Write to a number of PVs which may be scalars or arrays of different dimensions.
    • lcaPutNoWait — lcaPutNoWait is a variant of lcaPut that does not wait for the channel access put request to complete on the server prior to returning control to the command line.
    • lcaSetMonitor — Set a ``monitor'' on a set of PVs.
    • lcaSetSeverityWarnLevel — Set the warning threshold for lcaGet() operations.
